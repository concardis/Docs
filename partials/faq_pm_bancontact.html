Overview


The Bancontact debit card (formerly also know as Mister Cash) is a popular payment method in Belgium and the most commonly used card in that country. It lets customers make real-time card transactions to buy products and services. A Bancontact card with a PIN number is issued to customers who open a Belgian bank account. It’s linked to a customer’s current account and is accepted in most retail stores in Belgium.



CUSTOMER EXPERIENCE


  1. Customer selects Bancontact payment method in checkout
  2. Customer is redirected to a Bancontact page
  3. Customer decides how to pay
    1. Payment via Bancontact app
      1. Customer scans a QR-code via his Bancontact app and confirms the payment
      2. Customer opens his Bancontact app if he is shopping with his mobile device and confirms the payment
    2. Payment via Bancontact card
      1. Customer enters the PAN and expiration date
      2. Customer is asked to authenticate via 3d secure
      3. Customer confirms the payment
  4. Customer is redirected back to shop
    1. Shop should receive IBAN (at least if Bancontact card is used) which could be used for a refund

Technical Flow



Bancontact payment method is available trough PPRO Payment Platform. For this reason, the merchant must have his own PPRO account.


NEEDED PPRO SETTINGS


Field Description
Contract ID should be provided by the PPRO
Shared Secret should be provided by the PPRO
Notification Secret should be provided by the PPRO

Once the Customer chooses Bancontact payment, he will be transferred to Bancontact login page to proceed with the payment. The debit causes the immediately charging of the full amount.


As it is not ensured at the time of the redirect to the shop that the payment was 100% successful the Payengine waits for the final asynchronous notification from the bank before a status is moved from PENDING into SUCCESS or FAILURE.


Webhook notifications


A merchant should register for the Payengine webhook notifications in order to receive this final status update. A GET on the order or DEBIT transaction would also work, but it is not ensured that the final status will be available immediately after the redirect and a merchant might have to query the GET multiple times. Therefore it is strongly recommended that the asynchronous notifications from the Payengine are used insetad.


The Bancontact notifications could be subscribed

  • via the payengine API (Link to API documentation)
  • via the Payengine merchant center (Link to Merchant Center notifications documentation)

The following events have to be subscribed for in order to receive the Bancontact notifications, depending on the preferred source of notification.


Source Event
API notificationState = UPDATED
Merchant Center
event type = order update success
event type = order update failure

Refund transaction


If the transaction is with status SUCCESS, merchant can refund full or partial amount of the transaction.


Specifics


Bancontact input parameters


The following input parameters are required for initiating an Bancontact payment via direct API integration.


PARAMETER MANDATORY/OPTIONAL DESCRIPTION EXAMPLE
payment.accountHolder M Name of the bank account holder John Doe
async.successUrl M Redirect URL for successful payment http://my.shop.com/success
async.failureUrl M Redirect URL for unsuccessful payment http://my.shop.com/fail
async.cancelUrl M Redirect URL when payment is canceled http://my.shop.com/cancel
