Introduction


We offer a REST based API, with HTTP Basic Access Authorization and JSON as payload format.

Phylosophy

The API is designed to be human readable and easy to implement therefore tried and tested standards were used in developing the API. Developers should be empowered to implement business requirements in a pragmatic manner, whereas the product owners (merchants for example) have the flexibility to design products according to their vision.

System URLs and Endpoints

There are two main system URLs which are of interest:
  • https://api.payengine.de/v1
  • https://apitest.payengine.de/v1
api is the Live system, which will be used for real payments, whereas apitest is used for development and testing purposes. For more information on system URLs please go to System URLs.
